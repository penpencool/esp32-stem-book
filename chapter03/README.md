## 📖 บทนำ: ESP32-C3 — สมองเล็กที่ทำได้มหาศาล!

### 🤖 ลองนึกภาพตามนี้...

สมองของเราเป็น "ศูนย์กลางบัญชาการ" ที่ควบคุมร่างกายทั้งหมด:

- 🧠 สั่งให้มือขยับ
- 👀 สั่งให้ตาเห็น
- 💓 สั่งให้หัวใจเต้น

**ESP32-C3** ก็เป็น "สมองเล็กๆ" ที่ทำหน้าที่คล้ายกันสำหรับอุปกรณ์อิเล็กทรอนิกส์! มันคือ **Microcontroller** หรือ "ไมโครคอนโทรลเลอร์" — ชิปเล็กๆ ที่สามารถ:

- 📋 รับข้อมูลจาก "อวัยวะ" ต่างๆ (เช่น ปุ่ม, เซ็นเซอร์)
- 🧠 ประมวลผลตามโค้ดที่เราเขียน
- 📢 สั่งการ "อวัยวะ" ต่างๆ (เช่น LED, มอเตอร์, ลำโพง)

ESP32-C3 พิเศษตรงที่มันมี **Wi-Fi และ Bluetooth** ในตัว! เหมือนกับที่สมองมีเส้นประสาทหลายเส้น ทำให้ ESP32-C3 สามารถ "คุย" กับอุปกรณ์อื่นๆ ผ่านสัญญาณไร้สายได้! 📶

## 🔧 บอร์ด ESP32-C3 ดูยังไง?

### 🖼️ มารู้จักหน้าตาของ ESP32-C3

บอร์ด ESP32-C3 มีหลายรุ่นหลายยี่ห้อ แต่ส่วนใหญ่จะมีลักษณะคล้ายๆ กัน:

```
                    ┌─────────────────────┐
                    │   ┌───┐             │
                    │   │ESP│ LED สีแดง    │
                    │   │C3 │ (Power)      │
                    │   └───┘             │
                    │  ┌──────────┐      │
                    │  │  CHIP    │      │
                    │  │ESP32-C3  │  🔵  │
                    │  │ WROOM    │ (USB-C)
                    │  └──────────┘      │
                    │                     │
                    │ [==] [==] [==] [==] │
                    │  GPIO 0-3          │
                    │                     │
                    │ [==] [==] [==] [==] │
                    │  GPIO 4-7          │
                    └─────────────────────┘
```

#### 📌 ส่วนประกอบสำคัญของบอร์ด ESP32-C3:

| ส่วน | หน้าที่ |
|------|--------|
| **ESP32-C3 Chip** | "สมอง" หลักที่ประมวลผลทุกอย่าง |
| **USB-C / Micro USB** | ช่องเสียบสาย USB เพื่อต่อกับคอมพิวเตอร์และจ่ายไฟ |
| **LED Power** | ไฟสีแดง/เขียวติดเมื่อบอร์ดมีไฟ |
| **LED ที่ขา GPIO** | LED ที่ต่อกับขาบางขา (ขึ้นกับบอร์ด) |
| **ขา GPIO** | ขาสำหรับต่อวงจรอิเล็กทรอนิกส์ |
| **3V3 Pin** | ขาจ่ายไฟ 3.3 โวลต์ |
| **GND Pin** | ขากราวด์ (ไฟฟ้าลงดิน) |
| **RESET Button** | ปุ่มรีเซ็ตบอร์ด |
| **BOOT Button** | ปุ่มเข้าโหมดดาวน์โหลดโค้ด |

> 💡 **เคล็ดลับ:** หน้าตาของบอร์ดอาจดูต่างกันบ้างตามยี่ห้อ (เช่น ESP32-C3-DevKitM-1, ESP32-C3-WROOM, หรือบอร์ดจีนราคาถูก) แต่ "สมอง" (Chip) เป็นตัวเดียวกัน ดังนั้นโค้ดเดียวกันใช้ได้กับทุกบอร์ด!

### 📋 ตารางขาของ ESP32-C3 (GPIO)

**GPIO** ย่อมาจาก **General Purpose Input/Output** — แปลเป็นไทยว่า "ขาอินพุต/เอาต์พุต ทั่วไป" คือขาที่เราใช้ต่อวงจรและควบคุมได้ตามใจเรา!

```
    ╔═══════════════════════════════════════════════════════╗
    ║                    ESP32-C3 PINOUT                     ║
    ╠═══════════════════════════════════════════════════════╣
    ║                                                        ║
    ║  [3V3]  [GND]  [GPIO 0]  [GPIO 1]  [GPIO 2]  [GPIO 3]  ║
    ║    │      │        │         │         │         │    ║
    ║   Power  Ground    TX      RX      Flash    Strapping║
    ║                                                        ║
    ║  [GPIO 4]  [GPIO 5]  [GPIO 6]  [GPIO 7]  [GPIO 8]     ║
    ║       │         │         │         │         │     ║
    ║    SPI         SPI        ADC       ADC       ADC   ║
    ║                                                        ║
    ║  [GPIO 9]  [GPIO 10]  [GPIO 18]  [GPIO 19]  [GPIO 20] ║
    ║       │         │         │         │         │    ║
    ║    SPI        SPI       USB        USB       SPI    ║
    ║   (MOSI)      (MISO)              (D-)      (SCK)     ║
    ║                                                        ║
    ║  [GPIO 21]  [EN]                                                  ║
    ║       │        │                                                 ║
    ║    I2C      Enable                                              ║
    ║                                                        ║
    ╚═══════════════════════════════════════════════════════╝
```

#### 📌 GPIO ที่ใช้บ่อยสำหรับ ESP32-C3:

| GPIO | หน้าที่พิเศษ | หมายเหตุ |
|------|-------------|---------|
| **GPIO 0** | LED บิลด์อิน | LED ติดที่ขานี้ |
| **GPIO 1** | UART TX | ส่งข้อมูลออก |
| **GPIO 2** | ขาทั่วไป | เหมาะต่อ LED |
| **GPIO 3** | ขาทั่วไป | เหมาะต่อสวิตช์ |
| **GPIO 4** | ADC (วัดไฟ) | อ่านค่าอะนาล็อก |
| **3V3** | จ่ายไฟ 3.3V | ขาจ่ายไฟให้อุปกรณ์ |
| **GND** | กราวด์ | ขาลงดิน |
| **EN** | Enable | รีเซ็ต/เปิดบอร์ด |

> ⚠️ **ข้อควรระวัง:** บอร์ด ESP32-C3 จ่ายไฟที่ **3.3 โวลต์** ไม่ใช่ 5 โวลต์เหมือน Arduino! ถ้าต่ออุปกรณ์ที่ต้องการ 5V อาจเสียหายได้!

## ⚡ เรียนรู้เรื่องไฟฟ้าเบื้องต้น

### 🔹 ไฟฟ้าคืออะไร?

**ไฟฟ้า** คือ "กระแสอิเล็กตรอน" ที่ไหลผ่านสายไฟ ลองนึกภาพเป็น "น้ำ" ที่ไหลในท่อ:

- 💧 **น้ำ** = **ไฟฟ้า** (กระแส electrons)
- 🚿 **ความดันน้ำ** = **โวลต์ (Voltage)** — ทำให้น้ำไหล
- 🌊 **อัตราการไหลของน้ำ** = **กระแส (Current)** — ปริมาณน้ำที่ไหลผ่าน

### 🔹 โวลต์ (Voltage) คืออะไร?

**โวลต์ (V)** คือ "แรงดันไฟฟ้า" หรือ "ความดัน" ที่ผลักให้ไฟฟ้าไหล

#### 🎯 จำง่ายๆ:

```
🔋 แบตเตอรี่ AA = 1.5 โวลต์  → ผลักไฟฟ้าน้อย
🔋 แบตเตอรี่ 9V = 9 โวลต์    → ผลักไฟฟ้ามากขึ้น
⚡ ไฟบ้าน = 220 โวลต์       → ผลักไฟฟ้าเยอะมาก! (อันตราย!)
🔌 ESP32-C3 = 3.3 โวลต์     → ผลักไฟฟ้าต่ำ ปลอดภัย
```

#### 📏 สูตรความสัมพันธ์:

```
โวลต์ (V) = กระแส (I) × ความต้านทาน (R)
หรือเขียนสั้นๆ: V = I × R
```

> 💡 **เคล็ดลับ:** โวลต์เหมือน "ความดันน้ำ" — ถ้าความดันสูง น้ำจะไหลแรงและเร็ว แต่ถ้าสูงเกินไป ท่อก็อาจแตก! เหมือนกับวงจรไฟฟ้า ถ้าโวลต์สูงเกิน อุปกรณ์ก็เสียหายได้

### 🔹 กระแส (Current) คืออะไร?

**กระแส (Current, I)** คือ "ปริมาณไฟฟ้าที่ไหล" ในหนึ่งหน่วยเวลา มีหน่วยเป็น **แอมป์ (A)** หรือ **มิลลิแอมป์ (mA)**

#### 🎯 จำง่ายๆ:

```
💡 LED เล็ก ใช้กระแสประมาณ 20 mA (0.02 แอมป์)
⚡ มอเตอร์เล็ก ใช้กระแสประมาณ 100-500 mA
⚡ ESP32-C3 รวมทั้งบอร์ด ใช้กระแสประมาณ 50-150 mA
⚡ มือถือชาร์จ ใช้กระแสประมาณ 1-3 แอมป์
```

> ⚠️ **ข้อควรระวัง:** ถ้ากระแสไหลผ่าน "มากเกินไป" อุปกรณ์จะร้อนและเสียหายได้! นั่นเป็นเหตุผลว่าทำไมเราต้องใช้ **Resistor (ตัวต้านทาน)** เพื่อจำกัดกระแสให้เหมาะสม

### 🔹 ตัวต้านทาน (Resistor) คืออะไร?

**Resistor** คือ "ตัวต้านทาน" ทำหน้าที่ "ต้าน" การไหลของไฟฟ้า ทำให้กระแสลดลง มีหน่วยเป็น **โอห์ม (Ω)**

#### 🎯 จำง่ายๆ:

```
🔴🔴⚫⚫⚫ = Resistor 220Ω (แดง แดง ดำ ดำ น้ำตาล = 220)
🔴🔴⚫⚫⚫ = Resistor 220Ω (แดง แดง ดำ ดำ น้ำตาล = 220)
🔵🔵⚫⚫⚫ = Resistor 1kΩ (น้ำตาล�อย น้ำตาลลอย ดำ ดำ น้ำตาลลอย = 1000)
🔵⚫⚫⚫⚫ = Resistor 1MΩ (น้ำตาลลอย ดำ เขียว = 1 ล้าน)
```

```
สี | ค่า | สี | ค่า | สี | ค่า
-------------------------
ดำ | 0   | น้ำตาลลอย | 1M | ทอง | x0.1
น้ำตาล | 1   | เขียว | 10M | เงิน | x0.01
แดง | 2   | น้ำตาล | 100 | (ไม่มี) | 
ส้ม | 3   | แดง | 1k  |        |
เหลือง | 4   | น้ำตาลลอย | 10k |
เขียว | 5   | เขียว | 100k|
น้ำตาลลอย | 6   | น้ำตาล | 1M  |
น้ำตาลลอย | 7   |        |     |
เทา | 8   |        |     |
ขาว | 9   |        |     |
```

#### 📐 สูตรคำนวณค่า Resistor:

```
V = I × R

ถ้าเราต้องการ:
- โวลต์ (V) = ค่าที่เรารู้
- กระแส (I) = ค่าที่ LED ต้องการ (ปกติ 20mA = 0.02A)
- ความต้านทาน (R) = V/I
```

#### 📝 ตัวอย่าง: คำนวณ Resistor สำหรับ LED

สมมติเราต่อ LED กับ ESP32-C3 (3.3V):

```
ข้อมูล LED: ต้องการ 2V และ 20mA
แหล่งจ่ายไฟ: 3.3V (จาก ESP32-C3)

คำนวณ:
V ที่ตกคร่อม Resistor = 3.3V - 2V = 1.3V
R = V/I = 1.3V / 0.02A = 65Ω

ดังนั้น Resistor ที่เหมาะสมคือ 100Ω หรือ 220Ω (ปกติใช้ 220Ω)
```

> 💡 **เคล็ดลับ:** ถ้าไม่มี Resistor ค่าที่คำนวณได้ สามารถใช้ค่าที่ **มากกว่า** ได้ LED จะสว่างน้อยลงแต่ไม่เสียหาย!

## 🔗 วงจรอนุกรม vs วงจรขนาน

### 🔹 วงจรอนุกรม (Series Circuit)

**วงจรอนุกรม** คือการต่ออุปกรณ์ "เรียงต่อกัน" เหมือนต่อท่อน้ำต่อกันไปเรื่อยๆ

```
    ┌──── Resistor ────┐
    │                   │
────┤                   ├────
(+) │                   │ (-) Battery
    │                   │
    └───── LED ────────┘
```

#### 📌 คุณสมบัติของวงจรอนุกรม:

| คุณสมบัติ | สูตร | อธิบาย |
|----------|------|--------|
| **กระแส** | I รวม = I₁ = I₂ = ... | กระแสไหลผ่านทุกจุดเท่ากัน |
| **โวลต์** | V รวม = V₁ + V₂ + ... | โวลต์รวม = ผลบวกของทุกจุด |
| **ความต้านทาน** | R รวม = R₁ + R₂ + ... | ความต้านทานรวม = ผลบวก |
| **ข้อดี** | ต่อง่าย, ประหยัดสาย | ประหยัดสายไฟ |
| **ข้อเสีย** | ถ้าอุปกรณ์ตัวหนึ่งเสีย ทั้งวงจรหยุด | ถ้าอุปกรณ์ตัวใดตัวหนึ่งขาด กระแสไม่ไหล |

#### 📝 ตัวอย่าง: LED 2 ดวงต่ออนุกรม

```
โวลต์รวม = 3.3V
LED แต่ละดวง = 2V
LED 2 ดวงรวม = 4V > 3.3V ❌ ไม่พอ!

ดังนั้น LED 2 ดวงต่ออนุกรมกับ ESP32-C3 ไม่ได้ 
(เพราะโวลต์ไม่พอ)
```

### 🔹 วงจรขนาน (Parallel Circuit)

**วงจรขนาน** คือการต่ออุปกรณ์ "แยกกันไป" เหมือนท่อน้ำแยกออกเป็นหลายทาง

```
    ┌──── Resistor 1 ──── LED 1 ────┐
    │                               │
────┤                               ├────
(+) │                               │ (-) Battery
    │                               │
    └─── Resistor 2 ──── LED 2 ────┘
```

#### 📌 คุณสมบัติของวงจรขนาน:

| คุณสมบัติ | สูตร | อธิบาย |
|----------|------|--------|
| **โวลต์** | V รวม = V₁ = V₂ = ... | โวลต์ทุกจุดเท่ากัน |
| **กระแส** | I รวม = I₁ + I₂ + ... | กระแสรวม = ผลบวกของแต่ละเส้น |
| **ความต้านทาน** | 1/R = 1/R₁ + 1/R₂ + ... | ความต้านทานรวมลดลง |
| **ข้อดี** | ถ้าตัวหนึ่งเสีย ตัวอื่นยังทำงาน | แยกควบคุมได้ |
| **ข้อเสีย** | ใช้สายมากขึ้น | ต่อยากขึ้นเล็กน้อย |

#### 📝 ตัวอย่าง: LED 2 ดวงต่อขนาน

```
โวลต์รวม = 3.3V
LED แต่ละดวง = 2V
LED 2 ดวงขนาน = 3.3V ✓ พอดี!

ดังนั้น LED 2 ดวงต่อขนานกับ ESP32-C3 ได้
แต่ละดวงต้องมี Resistor ของตัวเอง
```

> 💡 **เคล็ดลับ:** สำหรับโปรเจกต์เริ่มต้น แนะนำให้ต่อวงจรขนาน เพราะถ้า LED ตัวหนึ่งเสีย ตัวอื่นยังทำงานได้!

## 🔌 ขา GPIO — Input vs Output

### 🔹 GPIO คืออะไร?

**GPIO (General Purpose Input/Output)** คือ "ขาทั่วไป" ที่เราสามารถ:

- **ตั้งเป็น Output** → ส่งสัญญาณออกไป (เช่น สั่ง LED ติด)
- **ตั้งเป็น Input** → รับสัญญาณเข้ามา (เช่น อ่านปุ่มกด)

### 🔹 Output Mode (ขาออก)

**Output** คือการตั้งให้ขา GPIO "ส่งสัญญาณออก" ไปควบคุมอุปกรณ์

#### 📌 การใช้งาน Output:

```cpp
pinMode(GPIO_NUMBER, OUTPUT);  // ตั้งค่าขาเป็น Output
digitalWrite(GPIO_NUMBER, HIGH); // ส่งไฟ 3.3V ออก (LED ติด)
digitalWrite(GPIO_NUMBER, LOW);  // ไม่ส่งไฟออก (LED ดับ)
```

#### 📝 ตัวอย่าง: ควบคุม LED

```cpp
#define LED_PIN 2  // กำหนดให้ GPIO 2 ควบคุม LED

void setup() {
  pinMode(LED_PIN, OUTPUT);  // ตั้ง GPIO 2 เป็นขาออก
}

void loop() {
  digitalWrite(LED_PIN, HIGH);  // สั่งให้ LED ติด
  delay(1000);                   // รอ 1 วินาที
  digitalWrite(LED_PIN, LOW);   // สั่งให้ LED ดับ
  delay(1000);                   // รอ 1 วินาที
}
```

### 🔹 Input Mode (ขาเข้า)

**Input** คือการตั้งให้ขา GPIO "รับสัญญาณเข้ามา" จากอุปกรณ์ภายนอก

#### 📌 การใช้งาน Input:

```cpp
pinMode(GPIO_NUMBER, INPUT);       // ตั้งค่าขาเป็น Input
int value = digitalRead(GPIO_NUMBER);  // อ่านค่า 0 หรือ 1

// value = 0 (LOW) → ไม่มีไฟเข้ามา
// value = 1 (HIGH) → มีไฟเข้ามา
```

#### 📝 ตัวอย่าง: อ่านสถานะปุ่มกด

```cpp
#define BUTTON_PIN 3  // ต่อปุ่มที่ GPIO 3

void setup() {
  Serial.begin(115200);
  pinMode(BUTTON_PIN, INPUT);  // ตั้ง GPIO 3 เป็นขาเข้า
}

void loop() {
  int buttonState = digitalRead(BUTTON_PIN);  // อ่านค่าปุ่ม
  
  if (buttonState == HIGH) {
    Serial.println("ปุ่มถูกกด!");  // ถ้ากด → พิมพ์ข้อความ
  } else {
    Serial.println("ปุ่มไม่ได้กด");  // ถ้าไม่กด
  }
  
  delay(100);  // รอเล็กน้อย
}
```

### 🔹 INPUT_PULLUP — ปุ่มกดแบบง่าย

ถ้าใช้ Input ธรรมดา ขา GPIO จะ "ลอย" ไม่มีค่าชัดเจนเมื่อปุ่มไม่ได้กด เราจึงใช้ **INPUT_PULLUP** เพื่อให้มีค่าเริ่มต้นเป็น HIGH

```cpp
pinMode(BUTTON_PIN, INPUT_PULLUP);  // ใช้ Pull-up Resistor ภายใน

// เมื่อกดปุ่ม → ไฟลง GND → อ่านได้ LOW
// เมื่อไม่กด → อ่านได้ HIGH
```

> 💡 **เคล็ดลับ:** INPUT_PULLUP ใช้ Resistor ภายในของ ESP32 ทำให้เราไม่ต้องต่อ Resistor ภายนอกเพิ่ม!

### 📝 ขั้นตอนที่ 1: รู้จักอุปกรณ์แต่ละตัว

#### LED (Light Emitting Diode)

4. **ต่อสายจากขาสั้น (Cathode)** ของ LED ไปยัง GND ของ ESP32-C3

### 📝 ขั้นตอนที่ 3: ต่อวงจรสวิตช์ (Input)

    │                                                         │
    │   ESP32-C3              Breadboard                      │
    │                                                         │
    │   ╔════════╗      ┌───────┐                            │
    │   ║  3V3   ║──────┤       │                            │

    │                                                         │

    │   ║  GND   ║──────┐ (ถ้าใช้ INPUT_PULLUP ต่อ GND ด้วย) │

#### ขั้นตอนการต่อสวิตช์ (แบบ INPUT_PULLUP):

1. **ต่อสายจาก 3V3** ไปยังขาข้างหนึ่งของสวิตช์
2. **ต่อสายจาก GPIO 3** ไปยังขาอีกข้างหนึ่งของสวิตช์
3. **ต่อสายจาก GND** ไปยังขาอีกข้างหนึ่งของสวิตช์ (ร่วมกับ GPIO 3 หรือต่อขาเดียวกัน)

> 💡 **เคล็ดลับ:** ในโหมด INPUT_PULLUP เมื่อกดสวิตช์จะอ่านได้ LOW เมื่อปล่อยจะอ่านได้ HIGH

### 📝 ขั้นตอนที่ 4: เขียนโค้ด

เปิด PlatformIO แล้วสร้าง Project ใหม่ หรือเขียนใน main.cpp:

```cpp
// ============================================
// บทที่ 3: LED ติด-ดับ ด้วยสวิตช์
// ============================================
// กดปุ่ม → LED ติด
// ปล่อยปุ่ม → LED ดับ

#define LED_PIN 2       // LED ต่อที่ GPIO 2
#define BUTTON_PIN 3    // สวิตช์ต่อที่ GPIO 3

void setup() {
  Serial.begin(115200);                // เริ่ม Serial
  pinMode(LED_PIN, OUTPUT);           // ตั้ง LED เป็นขาออก
  pinMode(BUTTON_PIN, INPUT_PULLUP); // ตั้งสวิตช์เป็นขาเข้า (Pull-up)
  
  Serial.println("=================================");
  Serial.println("  LED + Switch Test Started!");
  Serial.println("=================================");
  Serial.println("กดปุ่มเพื่อเปิด LED...");
  Serial.println();
}

void loop() {
  // อ่านค่าสถานะสวิตช์
  int buttonState = digitalRead(BUTTON_PIN);
  
  // ถ้ากดสวิตช์ (buttonState = LOW) → LED ติด
  // ถ้าปล่อยสวิตช์ (buttonState = HIGH) → LED ดับ
  digitalWrite(LED_PIN, !buttonState);  // ! คือ "not" กลับค่า
  
  // แสดงผลใน Serial Monitor
  if (buttonState == LOW) {
    Serial.println("🔵 สวิตช์ถูกกด → LED ติด");
  } else {
    Serial.println("⚪ สวิตช์ไม่ได้กด → LED ดับ");
  }
  
  delay(100);  // รอเล็กน้อย (100 มิลลิวินาที)
}
```

### 📝 ขั้นตอนที่ 5: อัปโหลดและทดสอบ

1. **เสียบสาย USB** ต่อ ESP32-C3 กับคอมพิวเตอร์
2. **เลือก Port** ถูกต้อง
3. **กด Upload** (ปุ่ม →) หรือ `Ctrl + Alt + U`
4. **เปิด Serial Monitor** ด้วยความเร็ว 115200
5. **ทดสอบ:** กดสวิตช์ → LED ควรติด, ปล่อย → LED ควรดับ

> 💡 **เคล็ดลับ:** ถ้า LED ติดกลับกัน (กดดับ ปล่อยติด) ให้เปลี่ยนโค้ดจาก `!buttonState` เป็น `buttonState`

1. LED 2 ดวงต่อ **อนุกรม** กับแบตเตอรี่

### แบบฝึกที่ 3: คำนวณค่า Resistor

ถ้าเราต่อ LED กับ ESP32-C3 (3.3V):
- LED ต้องการ 2V, 20mA
- ค่า Resistor ที่เหมาะสมคือเท่าไหร่?

<details>
<summary>✅ เฉลย (กดดู)</summary>

```
V ที่ต้องลด = 3.3V - 2V = 1.3V
R = V/I = 1.3V / 0.02A = 65Ω
ค่าที่ใช้ได้: 100Ω หรือ 220Ω
```
</details>

### แบบฝึกที่ 4: แก้ไขโค้ด

โค้ดนี้มีข้อผิดพลาด 3 จุด จงหาและแก้ไข:

void setup() {
  pinMode(LED, OUTPUT);  // จุดที่ 1
  pinMode(BUTTON, INPUT); // จุดที่ 2

  int value = digitalRead(BUTTON);

## 📚 สรุป

ในบทนี้เราได้เรียนรู้ว่า:

✅ **ESP32-C3** เป็น Microcontroller ที่มี Wi-Fi และ Bluetooth ในตัว มีขา GPIO หลายขาที่ใช้ต่ออุปกรณ์ได้

✅ **โวลต์ (Voltage)** คือแรงดันไฟฟ้าที่ผลักให้กระแสไหล ส่วน **กระแส (Current)** คือปริมาณไฟฟ้าที่ไหลผ่าน

✅ **Resistor** ทำหน้าที่ต้านทานการไหลของไฟฟ้า ช่วยป้องกันไม่ให้ LED และอุปกรณ์เสียหาย

✅ **วงจรอนุกรม** ต่อเรียงกัน กระแสเท่ากันทุกจุด ส่วน **วงจรขนาน** แยกกันไป โวลต์เท่ากันทุกจุด

✅ **GPIO ในโหมด Output** ส่งสัญญาณออกควบคุมอุปกรณ์ เช่น LED ส่วน **GPIO ในโหมด Input** รับสัญญาณเข้ามา เช่น สวิตช์

✅ **INPUT_PULLUP** ใช้ Resistor ภายในช่วยให้อ่านค่าสวิตช์ได้ถูกต้องโดยไม่ต้องต่อ Resistor ภายนอก

> 🔮 **บทต่อไป:** ในบทถัดไปเราจะมาเขียนโค้ดควบคุม LED ให้กระพริบเป็นจังหวะต่างๆ เรียนรู้เรื่องการหน่วงเวลา (delay) และการสร้างลูกเล่นต่างๆ กัน!
