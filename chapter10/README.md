## 💻 เนื้อหา

### 🔹 Relay ทำงานยังไง? — สวิตช์วิเศษที่ ESP32 สั่งได้

#### 🤔 Relay คืออะไร?

**Relay** (รีเลย์) คือ "สวิตช์ไฟฟ้าแบบแม่เหล็ก" (Electromagnetic Switch) ลองนึกภาพง่ายๆ นะ:

> จินตนาการว่า Relay คือ "หุ่นยนต์มือกดสวิตช์" ขนาดจิ๋ว
>
> - เมื่อเราส่งไฟเล็กๆ เข้าไป → หุ่นยนต์กดสวิตช์
> - เมื่อเราหยุดส่งไฟ → หุ่นยนต์ปล่อยสวิตช์

**ทำไมต้องใช้ Relay?**
เพราะ ESP32 ส่งสัญญาณได้แค่ 3.3V แต่พัดลมหรือหลอดไฟในบ้านต้องการ 220V! ถ้าเราเสียบ ESP32 เข้ากับ 220V โดยตรง → **ESP32 จะไหม้และพังหมด! 🔥** (และอันตรายมาก!)

ดังนั้น Relay จึงทำหน้าที่เป็น "สวิตช์มือกดอัตโนมัติ" ที่:
- รับสัญญาณไฟเล็กๆ (3.3V) จาก ESP32
- แล้วสั่งเปิด-ปิด ไฟฟ้า 220V ได้อย่างปลอดภัย

#### 🔬 โครงสร้างภายใน Relay

ภายใน Relay จะประกอบด้วย:

```
┌─────────────────────────────────────────┐
│           โครงสร้าง Relay                 │
│                                          │
│   [ขดลวดแม่เหล็กไฟฟ้า] ← รับสัญญาณจาก    │
│         │    ESP32 (3.3V)               │
│         │                                │
│         ↓                                │
│   [แขนเหล็กเคลื่อนที่] ← ถูกดึงเมื่อมีไฟ    │
│         │                                │
│    ┌────┴────┐                           │
│    │         │                           │
│   [NO]     [NC]  ← ขั้วต่อไฟฟ้า 220V     │
│                                          │
└─────────────────────────────────────────┘
```

1. **ขดลวดแม่เหล็กไฟฟ้า (Coil):** เมื่อมีไฟ 5V ผ่านเข้ามา จะกลายเป็นแม่เหล็กดึงแขนเหล็ก
2. **แขนเหล็กเคลื่อนที่ (Armature):** ถูกดึงลงมาเมื่อแม่เหล็กทำงาน
3. **ขั้ว NO (Normally Open):** หน้าสัมผัสที่**เปิดอยู่**ตอนไม่มีไฟ → เมื่อ Relay ทำงาน → **ปิดวงจร**
4. **ขั้ว NC (Normally Closed):** หน้าสัมผัสที่**ปิดอยู่**ตอนไม่มีไฟ → เมื่อ Relay ทำงาน → **เปิดวงจร**

#### 📌 NO vs NC — อันไหนดีกว่ากัน?

| โหมด | ตอนไม่มีไฟ (Standby) | ตอน Relay ทำงาน |
|------|---------------------|------------------|
| **NO (Normally Open)** | วงจรเปิด ❌ | วงจรปิด ✅ |
| **NC (Normally Closed)** | วงจรปิด ✅ | วงจรเปิด ❌ |

**เลือกใช้ยังไง?**
- ถ้าต้องการให้อุปกรณ์**เปิดเมื่อ Relay ทำงาน** → ใช้ **NO**
- ถ้าต้องการให้อุปกรณ์**ปิดเมื่อ Relay ทำงาน** → ใช้ **NC**

💡 **เคล็ดลับ:** สำหรับงานส่วนใหญ่ เรามักใช้ **NO** เพราะถ้าไฟดับหรือ ESP32 พัง อุปกรณ์จะปลอดภัย (ไม่ทำงาน) ไม่ใช่ทำงานตลอด!

#### 🛡️ Optocoupler — ระบบป้องกัน ESP32

ถ้าเราดู Relay Module ที่ซื้อมาจะเห็นว่ามี **IC ตัวเล็กๆ** ติดอยู่ ตัวนั้นคือ **Optocoupler** (ออปโตคัปเปลอร์) หรือเรียกอีกอย่างว่า **Opto-Isolator**

**Optocoupler ทำหน้าที่อะไร?**

```
ESP32 (3.3V)                    Relay (5V)
    │                              │
    │   ┌─────────────┐           │
    └──→│ Optocoupler │──────────→│
        │   (IC กันไฟ)  │           │
        └─────────────┘           │
              ↓                   │
         แยกกันสนิท!              │
    (ไฟสองฝั่งไม่ปนกัน)            │
```

**ข้อดีของ Optocoupler:**
- แยกวงจรไฟต่ำ (3.3V จาก ESP32) ออกจากวงจรไฟสูง (5V หรือ 220V)
- ถ้าไฟฟ้าขัดข้อง → Optocoupler พังก่อน → **ESP32 รอด!**
- ป้องกันไฟย้อนกลับเข้า ESP32

```
📌 จุดสำคัญ - Relay:
- Relay = สวิตช์แม่เหล็กที่ ESP32 สั่งเปิด-ปิดได้
- NO = ปกติเปิด → Relay ทำงาน → ปิดวงจร
- NC = ปกติปิด → Relay ทำงาน → เปิดวงจร  
- Optocoupler = ตัวกันไฟปนกัน ป้องกัน ESP32 พัง
```

### 🔹 หลักความปลอดภัย 220V — สิ่งที่ต้องจำ!

> ⚠️ **คำเตือนอันดับ 1 ของบทนี้!** ⚠️
>
> ไฟฟ้า 220V ในบ้านเรา **ฆ่าคนได้!** อย่าเล่นกับมัน!

#### 🚨 ทำไม 220V ถึงอันตราย?

- กระแสไฟฟ้า 30mA (0.03A) ผ่านหัวใจ = หัวใจหยุดเต้น!
- ไฟบ้านเรา 220V สามารถผลักกระแสไฟฟ้าผ่านร่างกายได้ง่าย
- ยิ่งเปียกหรือเหงื่อท่วม → ยิ่งนำไฟดี → **ยิ่งอันตราย!**

#### ✅ กฎ 5 ข้อที่ต้องจำเป็นตัวเลข!

**กฎข้อ 1: ปิดไฟก่อนทำงานเสมอ! 🔌**
```
ขั้นตอน: ปิดสวิตช์ไฟ → ถอดปลั๊ก → ตรวจสอบด้วยไขควงวัดไฟ
```

**กฎข้อ 2: สวมถุงมือฉนวน! 🧤**
```
ซื้อถุงมือยางที่ใช้ในงานไฟฟ้า (Electrical Gloves) สีแดงหรือสีส้ม
แม้แต่ถุงมือที่มีรอยฉีกขาดเล็กน้อย → ห้ามใช้!
```

**กฎข้อ 3: ใช้อุปกรณ์ที่มีมาตรฐาน! ✅**
```
ซื้อ Relay, สายไฟ, ปลั๊กที่มีความแข็งแรงและมีเครื่องหมาย มอก. (มาตรฐานผลิตภัณฑ์อุตสาหกรรม)
สัญลักษณ์ 🔌 บนอุปกรณ์หมายความว่า "ปลอดภัย"
```

**กฎข้อ 4: ไม่ทำงานคนเดียว! 👥**
```
ถ้าดูแล้วมีคนช่วยดู จะได้ช่วยปิดไฟถ้าเกิดอุบัติเหตุ
และถ้าเกิดไฟฟ้าดูด → มีคนช่วยปั๊มหัวใจ CPR ได้ทันท่วงที
```

**กฎข้อ 5: ตรวจสอบ 3 ครั้งก่อนจ่ายไฟ! 👀**
```
1. สายไฟต่อถูกต้อง → ไม่หลุด, ไม่ชำรุด
2. ขั้วไฟ L/N ถูกตำแหน่ง → L=Line (ไฟ), N=Neutral (N)
3. มี Fuse หรือ Circuit Breaker ป้องกัน → ไม่งั้นไฟจะลุกไหม้!
```

#### 📝 ในบทนี้เราจะทำอะไร?

เนื่องจากไฟ 220V อันตรายมาก เราจึงจะใช้:
- **พัดลม DC 12V** แทนพัดลม 220V — ใช้ไฟต่ำ ปลอดภัยกว่า
- **Adapter 12V** แทนไฟบ้านโดยตรง
- แต่จะ**อธิบายหลักการ 220V** ให้เข้าใจ เพื่อเวลาไปใช้จริงจะได้ทำถูกต้อง!

```
💡 เคล็ดลับ - ถ้าอยากลองจริงๆ กับ 220V:
1. ทำในห้องเรียน/ชมรม มีครูกำกับ
2. ใช้ Relay Module ที่รองรับ 220V จริงๆ
3. สวมถุงมือยาง + รองเท้าหุ้มส้น
4. ตรวจสอบวงจรทุกครั้งก่อนจ่ายไฟ
5. มีสวิตช์ตัดไฟฉุกเฉินใกล้มือ
```

### 🔹 Buzzer — เสียง Buzzer ดังกว่าลำโพงยังไง?

#### 🤔 Buzzer กับ ลำโพง ต่างกันยังไง?

เวลาเราดูหนังในโรง มีเสียงดังมากจากลำโพงใหญ่ แต่เวลามีเสียงเตือนในรถยนต์ มันจะ "ปี๊บๆๆ" เสียงดังและกระชับ — นั่นคือ **Buzzer** นั่นเอง!

| | **Buzzer** 🔔 | **ลำโพง** 🔊 |
|--|------------|------------|
| **เสียง** | เสียง "ปี๊บๆๆ" หรือ "บี๊บ๊บ" โมโนโทน | เสียงเพลง บทสนทนา หลากหลาย |
| **โครงสร้าง** | มีแผ่นเหล็กสั่นสะเทือน | มีขดลวด + แม่เหล็ก + กรวย |
| **การทำงาน** | ไฟผ่าน → แผ่นเหล็กสั่น → เกิดเสียง | ไฟผ่าน → ขดลวดสั่น → ขับกรวย → เกิดเสียง |
| **ใช้งาน** | เตือนภัย, นาฬิกาปลุก, กริ่ง | ฟังเพลง, ดูหนัง |
| **ราคา** | ถูก (5-20 บาท) | แพงกว่า (100-1000+ บาท) |

#### 🔬 Buzzer ทำงานยังไง?

มี **2 แบบ** ที่เราต้องรู้:

**1. Active Buzzer (บูซเซอร์แอคทีฟ)**

```
┌──────────────────────────┐
│   Active Buzzer          │
│                          │
│  [IC สร้างเสียง] ← แค่จ่ายไฟ  │
│       ↓                   │
│  [แผ่นเหล็กสั่น] ← สั่นเอง!   │
│                          │
└──────────────────────────┘
```

- ภายในมี IC สร้างความถี่ให้เลย
- แค่จ่ายไฟ 5V เข้าไป → ดังทันที!
- ใช้ง่ายมาก เหมาะกับมือใหม่
- **ข้อเสีย:** เปลี่ยนระดับเสียงไม่ได้ (ตายตัว)

**2. Passive Buzzer (บูซเซอร์แพสซีฟ)**

```
┌──────────────────────────┐
│   Passive Buzzer         │
│                          │
│  [ไม่มี IC]              │
│       ↓                   │
│  [แผ่นเหล็กสั่น] ← ต้องส่ง  │
│                   สัญญาณเอง │
│                          │
└──────────────────────────┘
```

- ไม่มี IC ภายใน ต้องส่งสัญญาณ PWM ให้
- ใช้ `tone()` ใน Arduino → ปรับความถี่ได้
- สามารถเล่นเพลงได้! (เปลี่ยนความถี่ = เปลี่ยนโน้ตเพลง)
- **ข้อเสีย:** ต้องเขียนโค้ดเยอะกว่า

#### 🎵 ลองนึกภาพ

Active Buzzer เหมือน **นกขับขาน** — เมื่อเปิดสวิตช์ → ร้องเพลงเดิมตลอด

Passive Buzzer เหมือน **กีตาร์** — ดีดสายแต่ละครั้งได้เสียงต่างกัน

#### 💻 วิธีต่อ Buzzer กับ ESP32-C3

**Active Buzzer:**
```
ESP32-C3          Active Buzzer
GPIO 2  ──────────►  (+) ขา +
GND     ──────────►  (-) ขา -
```
(บางรุ่นไม่ต้องต่อ R เพราะมี R ภายในแล้ว)

**Passive Buzzer:**
```
ESP32-C3          Passive Buzzer
GPIO 2  ──────────►  (+) ขา +
GND     ──────────►  (-) ขา -
```

```
📌 จุดสำคัญ - Buzzer:
- Active Buzzer = มี IC สร้างเสียงในตัว → จ่ายไฟดังทันที
- Passive Buzzer = ต้องส่งสัญญาณ PWM → ปรับเสียงได้
- ESP32 สามารถใช้คำสั่ง tone() ควบคุม Passive Buzzer ได้
- เสียง Buzzer ดังกว่าลำโพงในการเตือนภัย เพราะความถี่คงที่ → หูมนุษย์ตอบสนองดี!
```

### 🔹 วิธีต่อ Relay กับ ESP32-C3

#### 🔌 วงจร Relay Module

Relay Module ที่เราจะใช้จะมี 3 ส่วนหลัก:

```
┌────────────────────────────────────────────┐
│           Relay Module (5V 1 Channel)      │
│                                            │
│  ┌──────────────────────────────────────┐  │
│  │ ส่วนควบคุม (ต่อกับ ESP32)            │  │
│  │                                      │  │
│  │   VCC ────────►  5V จาก Adapter/USB   │  │
│  │   GND ────────►  GND                 │  │
│  │   IN  ────────►  GPIO ที่ต้องการ      │  │
│  │  (ขา Signal)                         │  │
│  └──────────────────────────────────────┘  │
│                                            │
│  ┌──────────────────────────────────────┐  │
│  │ ส่วนสวิตช์ (ต่อกับอุปกรณ์ไฟฟ้า)       │  │
│  │                                      │  │
│  │   COM  ───────►  ขั้วกลาง (ไฟเข้า)    │  │
│  │   NO   ────────►  ขั้วเปิด (ไฟออก)    │  │
│  │   NC   ────────►  ขั้วปิด (ไฟออก)     │  │
│  └──────────────────────────────────────┘  │
│                                            │
└────────────────────────────────────────────┘
```

#### 🔧 วิธีต่อ Relay กับอุปกรณ์ 12V

**กรณี: ควบคุมพัดลม 12V**

```
Adapter 12V                    Relay
  (+)  ──────────────────► COM (ขั้วกลาง)
  
  (-)  ──────────────────► พัดลม (-)
  
พัดลม (+) ────────────────► NO (ขั้วเปิด)
```

**หลักการทำงาน:**
1. ไฟจาก Adapter 12V เข้า COM (ขั้วกลาง)
2. เมื่อ Relay ทำงาน → NO ปิดวงจร → ไฟไหลผ่านพัดลม → พัดลมหมุน!
3. เมื่อ Relay ปิด → NO เปิดวงจร → ไฟไม่ไหล → พัดลมหยุด!

#### ⚠️ ความปลอดภัย - ก่อนต่อวงจร!

```
⚠️ ตรวจสอบก่อนต่อไฟทุกครั้ง!
1. ปิด Adapter ก่อนต่อสาย
2. ดึงปลั๊กออกจากไฟบ้านก่อนต่อสาย
3. ตรวจสอบว่า + กับ - ไม่สลับกัน
4. ตรวจสอบว่าสายไม่หลุด หรือฉีกขาด
5. ใส่ถุงมือถ้าทำกับไฟ 220V!
```

💡 **หมายเหตุ:** ถ้าใช้ USB จ่ายไฟ 5V จาก ESP32 อาจไม่พอสำหรับ Relay + พัดลม → ใช้ Adapter 12V จ่ายไฟแยกจะดีกว่า

#### ส่วนที่ 2: ESP32 กับ Buzzer

```
ESP32-C3          Active Buzzer
────────          ─────────────
GPIO 4        ──► (+) ขา +
GND           ──► (-) ขา -
```

#### ส่วนที่ 3: Relay กับ พัดลม 12V

```
Adapter 12V           Relay          พัดลม
─────────────         ─────          ─────
(+) 12V         ──► COM (ขั้วกลาง)
(-) 12V         ──► (-) พัดลม
(+) พัดลม       ──► NO (ขั้วเปิด)
```

> ⚠️ **สำคัญ:** ตรวจสอบว่าสาย + และ - ไม่สลับกัน! ถ้าสลับ → พัดลมอาจหมุนถอยหลังหรือพัง!

### 💻 ขั้นตอนที่ 3: เขียนโค้ด

เปิด Arduino IDE แล้วพิมพ์โค้ดนี้:

```cpp
// บทที่ 10: ควบคุมพัดลม 12V ด้วย Relay + เสียงเตือน Buzzer
// 🎯 เมื่อกดปุ่ม → พัดลมเปิด + เสียงบี๊บ 1 ครั้ง
// 🎯 เมื่อปล่อยปุ่ม → พัดลมปิด

#define RELAY_PIN   2    // ขา GPIO 2 ควบคุม Relay
#define BUZZER_PIN  4    // ขา GPIO 4 ควบคุม Buzzer
#define BUTTON_PIN  0    // ขา GPIO 0 เป็นปุ่มกด (ใช้ BOOT button)

void setup() {
  Serial.begin(115200);
  
  // ตั้งค่าขา Output
  pinMode(RELAY_PIN, OUTPUT);
  pinMode(BUZZER_PIN, OUTPUT);
  
  // ตั้งค่าขา Input (ปุ่มกด)
  pinMode(BUTTON_PIN, INPUT_PULLUP);
  
  // ตั้งค่าเริ่มต้น
  digitalWrite(RELAY_PIN, LOW);  // Relay ปิด (พัดลมไม่หมุน)
  digitalWrite(BUZZER_PIN, LOW); // Buzzer ปิด
  
  Serial.println("=== ระบบควบคุมพัดลม 12V ===");
  Serial.println("กดปุ่ม BOOT เพื่อเปิดพัดลม");
}

void loop() {
  // อ่านค่าปุ่มกด (กด = LOW เพราะใช้ INPUT_PULLUP)
  int buttonState = digitalRead(BUTTON_PIN);
  
  if (buttonState == LOW) {  // ถ้ากดปุ่ม
    // เปิด Relay → พัดลมหมุน
    digitalWrite(RELAY_PIN, HIGH);
    Serial.println("พัดลม: เปิด ON ✅");
    
    // เสียงบี๊บ 1 ครั้ง (เป็นการยืนยัน)
    digitalWrite(BUZZER_PIN, HIGH);
    delay(200);
    digitalWrite(BUZZER_PIN, LOW);
    
    delay(500);  // รอให้ปุ่มหยุดกด (debounce)
  } else {  // ถ้าปล่อยปุ่ม
    // ปิด Relay → พัดลมหยุด
    digitalWrite(RELAY_PIN, LOW);
    Serial.println("พัดลม: ปิด OFF");
  }
  
  delay(50);  // หน่วงเล็กน้อย
}
```

## 🔨 ปฏิบัติ — ระบบเตือนภัย: อุณหภูมิ > 40°C → Relay เปิดพัดลม + Buzzer

### 🎯 เป้าหมาย

สร้างระบบเตือนภัยที่:
1. วัดอุณหภูมิด้วย DHT11 ตลอดเวลา
2. ถ้าอุณหภูมิเกิน 40°C → เปิดพัดลมระบายอากาศ + เสียงเตือนดัง!
3. ถ้าอุณหภูมิกลับมาปกติ (< 35°C) → ปิดพัดลมและเสียง

### 🔧 อุปกรณ์เพิ่มเติม

| อุปกรณ์ | จำนวน | หมายเหตุ |
|---------|-------|---------|
| DHT11 | 1 อัน | วัดอุณหภูมิ + ความชื้น |
| ตัวต้านทาน 10kΩ | 1 ตัว | Pull-up สำหรับ DHT11 |

### 🔌 วงจร DHT11

```

GPIO 10      ──► DATA (ขา 2)
GND           ──► GND (ขา 4)

* ขา DATA ต้องต่อ Pull-up 10kΩ ไปที่ 3V3 (ขา 1)
```

### 💻 เขียนโค้ด

```cpp
// บทที่ 10: ระบบเตือนภัยอุณหภูมิ
// 🎯 อุณหภูมิ > 40°C → เปิดพัดลม + เสียงเตือน
// 🎯 อุณหภูมิ < 35°C → ปิดพัดลม + เสียง

#include <DHT.h>  // ใช้งาน DHT Sensor

#define RELAY_PIN   2    // ขา GPIO 2 → Relay
#define BUZZER_PIN  4    // ขา GPIO 4 → Buzzer
#define DHT_PIN     10   // ขา GPIO 10 → DHT11 DATA
#define DHT_TYPE    DHT11  // รุ่น DHT11

// ค่าอุณหภูมิที่กำหนด
#define TEMP_HIGH   40.0  // อุณหภูมิสูงสุด (°C)
#define TEMP_LOW    35.0  // อุณหภูมิต่ำสุด (°C)

// สถานะของระบบ
bool isAlarmActive = false;  // ไม่ได้เปิดเตือนตอนเริ่มต้น

// สร้าง object DHT
DHT dht(DHT_PIN, DHT_TYPE);

void setup() {
  Serial.begin(115200);
  
  // ตั้งค่าขา Output
  pinMode(RELAY_PIN, OUTPUT);
  pinMode(BUZZER_PIN, OUTPUT);
  
  // ตั้งค่าเริ่มต้น → ทุกอย่างปิด
  digitalWrite(RELAY_PIN, LOW);  // พัดลมปิด
  digitalWrite(BUZZER_PIN, LOW); // Buzzer ปิด
  
  // เริ่มใช้งาน DHT11
  dht.begin();
  
  Serial.println("=== ระบบเตือนภัยอุณหภูมิ ===");
  Serial.println("วัดอุณหภูมิทุก 2 วินาที...");
  Serial.print("เงื่อนไข: > ");
  Serial.print(TEMP_HIGH);
  Serial.print("°C → เปิดพัดลม + เสียงเตือน");
  Serial.print(" | < ");
  Serial.print(TEMP_LOW);
  Serial.println("°C → ปิดทุกอย่าง");
  Serial.println();
}

void loop() {
  // อ่านค่าอุณหภูมิจาก DHT11
  float temperature = dht.readTemperature();
  
  // ตรวจสอบว่าอ่านค่าได้ถูกต้องหรือเปล่า
  if (isnan(temperature)) {
    Serial.println("❌ อ่านค่า DHT11 ไม่ได้! ตรวจสอบการต่อสาย");
    delay(2000);
    return;
  }
  
  // แสดงค่าอุณหภูมิใน Serial Monitor
  Serial.print("🌡️ อุณหภูมิ: ");
  Serial.print(temperature);
  Serial.println("°C");
  
  // ===== ตรวจสอบเงื่อนไข =====
  
  if (temperature > TEMP_HIGH) {
    // ✅ เงื่อนไขที่ 1: อุณหภูมิสูงเกิน → เปิดเตือนภัย
    if (!isAlarmActive) {
      Serial.println("⚠️⚠️⚠️ อุณหภูมิสูงเกิน! เปิดระบบเตือนภัย!");
      isAlarmActive = true;
    }
    
    // เปิดพัดลม (Relay ON)
    digitalWrite(RELAY_PIN, HIGH);
    Serial.println("  → พัดลม: เปิด ON 🔄");
    
    // เสียงเตือน (เล่น 3 ครั้ง)
    for (int i = 0; i < 3; i++) {
      digitalWrite(BUZZER_PIN, HIGH);
      delay(150);
      digitalWrite(BUZZER_PIN, LOW);
      delay(100);
    }
    Serial.println("  → เสียงเตือน: ดัง! 🔔🔔🔔");
    
  } else if (temperature < TEMP_LOW && isAlarmActive) {
    // ✅ เงื่อนไขที่ 2: อุณหภูมิกลับมาปกติ → ปิดเตือนภัย
    Serial.println("✅ อุณหภูมิกลับมาปกติ ปิดระบบเตือนภัย");
    
    // ปิดพัดลม (Relay OFF)
    digitalWrite(RELAY_PIN, LOW);
    Serial.println("  → พัดลม: ปิด OFF");
    
    // ปิดเสียง
    digitalWrite(BUZZER_PIN, LOW);
    Serial.println("  → เสียงเตือน: ปิด");
    
    isAlarmActive = false;
  }
  
  Serial.println();  // ขึ้นบรรทัดใหม่
  delay(2000);  // วัดอุณหภูมิทุก 2 วินาที
}
```

### 📤 อัปโหลดและทดสอบ

1. **กดปุ่ม Upload** (ลูกศรชี้ขวา)
2. **เปิด Serial Monitor** ที่ 115200 baud
3. **สังเกตค่าอุณหภูมิ** ที่แสดงทุก 2 วินาที

#### ทดสอบโดยการ:
- **เป่าลมร้อน**ไปที่ DHT11 (หรือใช้นิ้วแตะเบาๆ) → อุณหภูมิจะสูงขึ้น
- พอเกิน 40°C → พัดลมจะเปิด + เสียงเตือนดัง 3 ครั้ง!
- ปล่อยให้เย็นลง → พอต่ำกว่า 35°C → พัดลมและเสียงปิด

> 💡 **เคล็ดลับ:** ถ้าอยากทดสอบเร็ว ลองเปลี่ยนค่า `TEMP_HIGH` เป็น 30 และ `TEMP_LOW` เป็น 28 แล้วใช้นิ้วแตะ DHT11 ได้เลย!

## 🤔 คำถามท้ายบท

### คำถามที่ 1: อธิบายหลักการทำงานของ Relay
**Relay ทำงานยังไง?** เมื่อเราจ่ายไฟ 5V เข้าที่ขดลวด (Coil) ของ Relay จะเกิดสนามแม่เหล็กดึงแขนเหล็กเคลื่อนที่ ทำให้หน้าสัมผัส NO ปิดวงจร (หรือ NC เปิดวงจร) ผลลัพธ์คือเราสามารถใช้สัญญาณไฟเล็กๆ (3.3V จาก ESP32) ควบคุมการเปิด-ปิดวงจรไฟฟ้าแรงสูง (220V) ได้โดยไม่ต้องต่อไฟตรง!

### คำถามที่ 2: NO กับ NC ต่างกันยังไง?
**Normally Open (NO)** คือหน้าสัมผัสที่ปกติ (ตอนไม่จ่ายไฟให้ Relay) จะ**เปิดวงจรอยู่** พอ Relay ทำงาน → ปิดวงจร → ไฟไหล → อุปกรณ์ทำงาน

**Normally Closed (NC)** คือหน้าสัมผัสที่ปกติจะ**ปิดวงจรอยู่** พอ Relay ทำงาน → เปิดวงจร → ไฟไม่ไหล → อุปกรณ์หยุดทำงาน

สำหรับงานส่วนใหญ่ เราเลือก **NO** เพราะถ้าไฟดับหรือ ESP32 พัง → อุปกรณ์จะหยุดทำงาน (ปลอดภัยกว่า)

### คำถามที่ 3 (เชิงลึก): ทำไมต้องมี Optocoupler?
**Optocoupler ทำหน้าที่แยกวงจรไฟต่ำ (3.3V/5V) ออกจากวงจรไฟสูง (220V)**

ถ้าไม่มี Optocoupler:
- ไฟ 220V อาจย้อนเข้า ESP32 → เครื่องระเบิด!
- ความผิดพลาดเล็กน้อย → ESP32 พังทันที

มี Optocoupler:
- ไฟสองฝั่งแยกกันสนิท ไม่ปนกัน
- ถ้ามีปัญหา → Optocoupler พังก่อน → ESP32 รอด!

### คำถามที่ 4: Active Buzzer กับ Passive Buzzer ต่างกันยังไง?
**Active Buzzer** มี IC สร้างเสียงในตัว แค่จ่ายไฟ 5V เข้าไปก็ดังทันที ใช้งานง่ายมาก แต่เสียงคงที่ (ปรับไม่ได้)

**Passive Buzzer** ไม่มี IC ต้องส่งสัญญาณ PWM หรือใช้คำสั่ง `tone()` เพื่อสร้างเสียง ทำให้ปรับความถี่เสียงได้ → เล่นเพลงได้!

## 📚 สรุป

ในบทนี้เราได้เรียนรู้ว่า:

✅ **Relay** คือสวิตช์แม่เหล็กที่ ESP32 สั่งเปิด-ปิดไฟฟ้าแรงสูงได้  
✅ **NO (Normally Open)** คือหน้าสัมผัสที่ปกติเปิด → Relay ทำงาน → ปิดวงจร  
✅ **NC (Normally Closed)** คือหน้าสัมผัสที่ปกติปิด → Relay ทำงาน → เปิดวงจร  
✅ **Optocoupler** ช่วยแยกวงจรไฟต่ำ-สูง ป้องกัน ESP32 พัง  
✅ **ไฟฟ้า 220V อันตรายถึงชีวิต!** ต้องสวมถุงมือ ปิดไฟก่อนทำ และมีผู้ใหญ่ดูแล  
✅ **Buzzer** มี 2 แบบ: Active (จ่ายไฟดัง) และ Passive (ต้องส่งสัญญาณ ปรับเสียงได้)  
✅ สร้างระบบควบคุมพัดลม 12V และระบบเตือนภัยอุณหภูมิได้สำเร็จ!  

> 🔮 **บทต่อไป:** ในบทถัดไปเราจะมารวมทุกอย่างที่เรียนรู้มาเป็นโปรเจกต์ "สมาร์ทฟาร์ม" ขนาดเล็กกัน! วัดอุณหภูมิ วัดความชื้น รดน้ำอัตโนมัติ และแสดงผลบนจอ OLED กันเลย! 🌱
