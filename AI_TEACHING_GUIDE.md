# 📘 ESP32-C3 STEM AI Coding — คู่มือการสอนด้วย AI

## 🎯 วัตถุประสงค์

คู่มือนี้สำหรับครู/ผู้สอน และ AI ที่จะช่วยสอนนักเรียนเขียนโค้ดบน ESP32-C3 โดยใช้หลัก **AI-Augmented Learning** — ครูเป็นผู้กำหนดทิศทาง AI ช่วยอธิบายและเขียนโค้ด

## 🤖 หลักการใช้ AI สอน

### 1. AI คืออะไร? (สำหรับนักเรียน)

```
AI คือ "เพื่อนฉลาด" ที่:
- รู้เยอะมาก (เรียนรู้จากข้อมูลมหาศาล)
- ตอบเร็ว (ไม่ต้องเปิดหนังสือ)
- ช่วยเขียนโค้ดได้ (แปลความคิดเป็นโค้ด)
- แก้บักได้ (หาที่ผิดให้)
```

### 2. กฎทองในการใช้ AI

| ทำได้ ✅ | ทำไม่ได้ ❌ |
|-----------|--------------|
| ถาม AI ให้อธิบายโค้ด | ส่งโค้ดทั้งหมดให้ copy-paste |
| ให้ AI ช่วยแก้บัก | ถามคำตอบสอบโดยไม่ลองคิด |
| ขอให้ AI สร้างโครงโค้ด | ขอโค้ดเต็มแล้วส่งทันที |
| ถาม AI ให้แนะนำวิธีทำ | ขอให้ทำแทนทั้งหมด |

### 3. วิธีถาม AI ให้ได้คำตอบดี

#### ❌ ถามกว้างๆ (ได้คำตอบไม่ดี)
```
"ทำโปรเจกต์หุ่นยนต์ยังไง?"
```

#### ✅ ถามเฉพาะเจาะจง (ได้คำตอบดี)
```
"ช่วยเขียนโค้ด HC-SR04 วัดระยะบน ESP32-C3 และแสดงผลบน OLED 0.96 นิ้ว"
"อธิบายว่า pulseIn() ทำงานยังไง"
"โค้ดนี้ผิดตรงไหน: [paste code]"
```

## 📚 ความรู้พื้นฐานสำหรับ AI สอน

### 🧠 ระดับความรู้นักเรียน

```
มัธยมต้น (M.1-M.3):
- ยังไม่เคยเขียนโค้ด
- รู้พื้นฐานคอมพิวเตอร์
- ชอบเรียนรู้จากตัวอย่าง
- สนใจเรื่อง IoT/Smart Home

มัธยมปลาย (M.4-M.6):
- มีพื้นฐานการเขียนโค้ดบ้าง
- ต้องการความลึกมากขึ้น
- สนใจว่างานจริงใช้อะไร
```

### 📖 สิ่งที่ต้องสอนก่อนเริ่ม

#### ครั้งที่ 1: รู้จัก Computer (2 ชม.)

```
หัวข้อ:
1. คอมพิวเตอร์คืออะไย?
2. Hardware vs Software
3. โปรแกรมคืออะไร?
4. ทำไมต้องเขียนโค้ด?

กิจกรรม:
- ให้นักเรียนยกตัวอย่างโปรแกรมที่ใช้ทุกวัน
- เปรียบเทียบ: คนสั่งเชฟทำอาหาร vs เขียนโค้ดสั่งคอม
```

#### ครั้งที่ 2: รู้จัก ESP32-C3 (2 ชม.)

```
หัวข้อ:
1. ESP32-C3 คืออะไร?
2. MCU vs CPU vs GPU
3. GPIO คือขาที่ใช้คุยกับอุปกรณ์
4. Input vs Output

กิจกรรม:
- ดูบอร์ดจริง + ชี้ขาต่างๆ
- ทดลอง: กดสวิตช์ → LED ติด
```

## 🎓 เทคนิคการสอน

### 1. สอนโดยใช้ Project-Based Learning

```
แทนที่จะสอนทฤษฎีก่อน แล้วค่อยทำโปรเจกต์:

1. สาธิตโปรเจกต์ที่น่าสนใจก่อน (เช่น ไฟกระพริบ)
2. ให้นักเรียนเดาว่ามันทำงานยังไง
3. ค่อยอธิบายทฤษฎีที่เกี่ยวข้อง
4. ให้ลงมือทำตาม
5. ปรับปรุงโดยเพิ่มฟีเจอร์เอง
```

### 2. สอนโดยใช้ AI เป็น Tutor

```
รอบนี้: AI = ติวเตอร์ส่วนตัว

1. นักเรียนทำโจทย์ → ส่งให้ AI ตรวจ
2. AI บอกว่าถูก/ผิด + ชี้แนะ
3. นักเรียนแก้ไข → ส่งให้ AI อีกรอบ
4. วนจนสำเร็จ

ข้อดี: นักเรียนได้ลงมือทำเอง + ได้ feedback ทันที
```

### 3. สอนโดยให้ Debug บนโค้ดจริง

```
วิธี Debug สนุกๆ:

1. ครูเตรียมโค้ดที่มีบัก 3-4 จุด
2. ให้นักเรียนหาว่าบักอยู่ตรงไหน
3. ให้ AI ช่วยอธิบายว่าทำไมมันผิด
4. ให้แก้ไข + ทดสอบ

เช่น:
- ลืม semicolon
- ใช้ตัวแปรผิดชื่อ
- ต่อขาผิดในวงจร
```

## ⚠️ ข้อควรระวังในการสอน

### 1. ความปลอดภัยไฟฟ้า 220V

```
⚠️ ข้อห้ามเด็ดขาด:

1. ห้ามต่อวงจรขณะเปิดไฟ
2. ห้ามแตะสายไฟ bare wire
3. ห้ามทำในที่ชื้น
4. ต้องมีผู้ใหญ่ดูแลเสมอเมื่อต่อ 220V

✅ สิ่งที่ทำได้:
- ต่อวงจร LED/Relay/Servo 12V ได้เลย
- ต่อ 220V ได้แค่ผ่าน Relay เท่านั้น
```

### 2. อย่าให้ Copy-Paste โดยไม่เข้าใจ

```
❌ วิธีที่ไม่ดี:
"เอาโค้ดนี้ไปใช้เลย"

✅ วิธีที่ดี:
"ลองอ่านโค้ดทีละบรรทัด + ถาม AI ว่าบรรทัดนี้ทำอะไร
 พอเข้าใจแล้วค่อยกดอัปโหลด"
```

### 3. AI ก็มีผิดได้

```
AI บางครั้งก็ตอบผิด!
ดังนั้นต้อง:

1. ตรวจสอบคำตอบจาก AI เสมอ
2. ทดลองรันโค้ดดูว่าทำงานจริงมั้ย
3. ถ้าไม่แน่ใจ → ถามครู
```

## 📊 การประเมินผล

### เกณฑ์การให้คะแนน

| หัวข้อ | คะแนน | วิธีวัด |
|---------|--------|----------|
| ความเข้าใจทฤษฎี | 20% | ถาม-ตอบ + สอบเล็กๆ |
| ทักษะเขียนโค้ด | 30% | ทำโจทย์ + โปรเจกต์ |
| การใช้ AI ช่วย | 20% | ดูว่าใช้ได้ถูกต้อง |
| โปรเจกต์สุดท้าย | 30% | สร้างสิ่งของตัวเองได้ |

### ตัวอย่างการประเมิน

```
✅ ดีมาก (A):
- ทำโปรเจกต์ได้เอง 80%
- ใช้ AI ช่วยเป็น แต่เข้าใจที่ทำ
- อธิบายโค้ดตัวเองได้

✅ ดี (B):
- ทำโปรเจกต์ได้ 60%
- ต้องใช้ AI ช่วยเยอะ
- แต่เข้าใจว่าทำไมต้องทำแบบนี้

⚠️ ต้องปรับปรุง (C):
- Copy-paste โค้ดโดยไม่เข้าใจ
- ใช้ AI แต่ไม่ตรวจสอบ
- ทำโปรเจกต์ไม่เสร็จ
```
