## 💻 เนื้อหา

### 🔹 หลักการวัดระยะด้วย Ultrasound (Sonar) 🦇

#### 🎯 HC-SR04 ทำงานยังไง?

HC-SR04 เป็นเซ็นเซอร์วัดระยะด้วยอัลตร้าโซนิค (Ultrasonic) ประกอบด้วย 2 ส่วนหลัก:

1. **Transmitter (T)** — ตัวส่งเสียง Ultrasound ออกไป
2. **Receiver (R)** — ตัวรับเสียงที่สะท้อนกลับมา

```
    HC-SR04 (มองจากด้านหน้า)

         ┌─────────────────────┐
         │  ┌───┐    ┌───┐    │
         │  │ T │    │ R │    │
         │  └───┘    └───┘    │
         │   ส่ง       รับ      │
         │ Transmitter Receiver│
         └─────────────────────┘

    ขา (จากซ้ายไปขวา):
    VCC | Trig | Echo | GND
```

เมื่อเราสั่งให้ HC-SR04 วัดระยะ กระบวนการจะเป็นแบบนี้:

```
    ขั้นตอนที่ 1: ส่งสัญญาณ Trigger
    ──────────────────────────────
    ESP32 ────► HC-SR04 (Trig pin)
    ให้ส่งเสียง Ultrasound ออกไป

    ขั้นตอนที่ 2: Ultrasound ออกไป
    ──────────────────────────────
    HC-SR04 ส่งเสียง 8 ลูกคลื่นที่ 40kHz ออกไป
    (ความถี่นี้มนุษย์ไม่ได้ยิน แต่เหมาะกับการวัดระยะ)

    ขั้นตอนที่ 3: เสียงสะท้อนกลับ
    ──────────────────────────────
    เมื่อเสียงกระทบวัตถุ จะสะท้อนกลับมาหา HC-SR04
    (เหมือนเสียงตะโกนในถ้ำ!)

    ขั้นตอนที่ 4: Echo Pin ส่งสัญญาณกลับ
    ──────────────────────────────
    HC-SR04 (Echo pin) ────► ESP32
    ส่งสัญญาณ HIGH มาให้ตลอดเวลาที่รอเสียงสะท้อน
```

```
📌 จุดสำคัญ:
- HC-SR04 ส่งเสียง Ultrasound ความถี่ 40kHz
- ความถี่นี้มนุษย์ไม่ได้ยิน (หูมนุษย์ได้ยินแค่ 20Hz - 20kHz)
- HC-SR04 วัดระยะได้ 2cm - 400cm
- มุมวัดประมาณ 15 องศา
```

#### 🔬 ทำไมต้องเป็น Ultrasound?

เสียงที่เราได้ยิน (20Hz - 20kHz) มีความยาวคลื่นยาวเกินไป ทำให้สะท้อนได้ไม่ดีกับวัตถุเล็กๆ Ultrasound ที่ 40kHz มีความยาวคลื่นประมาณ 8.5mm ซึ่งเหมาะกับการสะท้อนจากวัตถุทั่วไปได้ดี

```
    เปรียบเทียบความยาวคลื่น:

    เสียงที่พูด (1kHz) → ความยาวคลื่น ≈ 34cm
    เสียง Ultrasound (40kHz) → ความยาวคลื่น ≈ 0.85cm

    วัตถุที่ใหญ่กว่าความยาวคลื่น → สะท้อนได้ดี
    วัตถุที่เล็กกว่าความยาวคลื่น → เสียงจะ "เลี้ยว" ผ่านไป
```

### 🔹 การคำนวณระยะทางจากเวลา (Time of Flight) 📐

#### ⏱️ Time of Flight (ToF) — เวลาบิน

หลักการคำนวณระยะทางง่ายมาก:

**ระยะทาง = ความเร็วเสียง × เวลา / 2**

ทำไมต้องหารด้วย 2? เพราะเสียงไปและกลับ!

```
    ESP32                          วัตถุ
       │                             │
       │── ส่งเสียง (Trigger) ──────►│
       │                             │
       │◄── รอเสียงสะท้อน (Echo) ───│
       │                             │

    เวลาทั้งหมด = เวลาไป + เวลากลับ
    ระยะทาง = (ความเร็วเสียง × เวลาทั้งหมด) / 2
```

#### 🔢 สูตรคำนวณ

**ความเร็วเสียงในอากาศ = 343 เมตร/วินาที (ที่ 20°C) = 0.0343 เซนติเมตร/ไมโครวินาที**

แต่เนื่องจาก ESP32 วัดเวลาเป็นไมโครวินาที (μs) เราจึงใช้สูตร:

```
    ระยะทาง (cm) = (เวลาในหน่วย μs × 0.0343) / 2

    หรือเขียนง่ายๆ ว่า:

    ระยะทาง (cm) = เวลา (μs) / 58
```

> 💡 **ทำไม 58?** เพราะ 1 / (2 × 0.0343) ≈ 58 นั่นเอง!

```
📌 ตัวอย่างการคำนวณ:

    ถ้า Echo Pin ส่งสัญญาณ HIGH มา 294 μs
    (นั่นคือเวลาที่เสียงใช้ไปและกลับ)

    ระยะทาง = 294 / 58 ≈ 5 cm
```

```
⚠️ ข้อควรระวัง:
- ความเร็วเสียงขึ้นอยู่กับอุณหภูมิ!
  ที่ 0°C = 331 m/s, ที่ 20°C = 343 m/s, ที่ 30°C = 349 m/s
- ถ้าต้องการความแม่นยำสูง ควรใช้สูตรปรับอุณหภูมิ:
  ความเร็วเสียง (m/s) = 331 + (0.6 × อุณหภูมิ °C)
```

#### 📊 ตารางเปรียบเทียบระยะกับเวลา

| ระยะทาง | เวลา (μs) |
|---------|----------|
| 2 cm | 116 μs |
| 5 cm | 294 μs |
| 10 cm | 588 μs |
| 20 cm | 1,176 μs |
| 50 cm | 2,940 μs |
| 100 cm | 5,880 μs |
| 200 cm | 11,760 μs |
| 400 cm | 23,520 μs |

### 🔹 วิธีต่อ HC-SR04 กับ ESP32-C3 🔌

#### 🔌 ขาของ HC-SR04

HC-SR04 มี 4 ขา:

| ขา | ชื่อ | หน้าที่ | เชื่อมต่อกับ |
|----|------|--------|------------|
| 1 | VCC | ไฟเลี้ยง (+5V) | USB/Adapter 5V |
| 2 | Trig | Trigger (สั่งส่งเสียง) | ESP32 GPIO (Output) |
| 3 | Echo | Echo (รับสัญญาณกลับ) | ESP32 GPIO (Input) |
| 4 | GND | Ground | GND ร่วม |

```
⚠️ คำเตือนสำคัญ:
- HC-SR04 ต้องใช้ไฟ 5V (VCC) จึงจะทำงานได้ปกติ
- Echo Pin จะส่งสัญญาณ 5V กลับมา → ต้องใช้ Voltage Divider หรือ
  ต่อผ่าน Logic Level Converter เพื่อไม่ให้ ESP32-C3 (3.3V) พัง!
- Trig Pin สามารถสั่งด้วย 3.3V ได้เลย (เป็น Input ของ HC-SR04)
```

#### 🔧 วิธีต่อ — มี 2 วิธี

**วิธีที่ 1: ใช้ Voltage Divider สำหรับ Echo Pin (แนะนำ)**

เนื่องจาก Echo Pin ส่งสัญญาณ 5V ออกมา แต่ ESP32-C3 รับได้สูงสุด 3.3V เราต้องลดแรงดันลง

```
    Voltage Divider Circuit:

    HC-SR04 Echo (5V) ──┤R1 1kΩ├──┬──► ESP32 GPIO (Echo)
                         │        │
                         └────────┤R2 2kΩ├──► GND
                                  │
                                  └──► GND

    สูตร: Vout = Vin × (R2 / (R1 + R2)) = 5V × (2000/3000) ≈ 3.3V ✅
    (ESP32-C3 รับ HIGH ที่ > 2.5V ดังนั้น 3.3V ถือว่าปลอดภัย)
```

> ⚠️ **หมายเหตุ:** ค่า R1=1kΩ, R2=2kΩ ให้ Vout = 3.33V ซึ่งเกิน 3.3V เล็กน้อย หากต้องการค่าที่ใกล้เคียง 3.3V มากขึ้น แนะนำ **R1=1kΩ, R2=2.2kΩ** (ได้ Vout ≈ 3.44V) หรือ **R1=1.5kΩ, R2=3kΩ** (ได้ Vout ≈ 3.33V ที่แม่นยำกว่า)

**วิธีที่ 2: ใช้ Logic Level Converter (TTL Converter)**

ใช้บอร์ด LLC ขนาดเล็กต่อระหว่าง 5V กับ 3.3V ได้เลย ใช้ง่ายและปลอดภัยกว่า

```
    ESP32-C3                    HC-SR04
    ┌─────────┐              ┌──────────┐
    │      GND├──────────────│○ GND     │
    │  3V3    ├──────────────│○ VCC (5V)│
    │ GPIO3  ├───────────────│○ Trig    │
    │ GPIO10◄─┤◄── Voltage ───│○ Echo    │
    │        │    Divider    │          │
    └─────────┘              └──────────┘

    Voltage Divider:
    R1 = 1kΩ (น้ำตาล-ดำ-แดง)
    R2 = 2kΩ (แดง-ดำ-แดง)
```

**สรุปการต่อสาย:**

| HC-SR04 | ESP32-C3 / อุปกรณ์ |
|--------|--------------------|
| VCC | 5V Power Supply |
| GND | GND ร่วม |
| Trig | GPIO 3 |
| Echo | → Voltage Divider (R1 1kΩ) → GPIO 10 |
| Echo | → Voltage Divider (R2 2kΩ) → GND |

## 🔨 ปฏิบัติ (พร้อมโค้ด)

### 🛠️ ปฏิบัติที่ 1: เครื่องวัดระยะแสดงผลบน OLED 📟

โปรเจกต์นี้เราจะสร้าง "เครื่องวัดระยะ" ที่แสดงระยะทางบนจอ OLED แบบเรียลไทม์ เหมือนเครื่องวัดระยะเลเซอร์ในงานก่อสร้างเลย! 📏

#### ขั้นตอนที่ 1: ต่อวงจร

```
    ESP32-C3                    HC-SR04
    ┌─────────┐              ┌──────────┐
    │      GND├──────────────│○ GND     │
    │  5V    ├──────────────│○ VCC     │
    │  GPIO3 ├──────────────│○ Trig    │
    │  GPIO10◄─┤◄── R1+R2 ───│○ Echo    │
    └─────────┘              └──────────┘

    ESP32-C3                    OLED (SSD1306)
    ┌─────────┐              ┌──────────┐
    │      GND├──────────────│○ GND     │
    │  3V3   ├──────────────│○ VCC     │
    │  GPIO1 ├──────────────│○ SCL     │ ← ⚠️ เปลี่ยนจาก GPIO 6
    │  GPIO0 ├──────────────│○ SDA     │ ← ⚠️ เปลี่ยนจาก GPIO 7
    └─────────┘              └──────────┘

    Voltage Divider สำหรับ Echo:
    HC-SR04 Echo ──┤1kΩ├──┬──► ESP32 GPIO10
                          │
                          └──┤2kΩ├──► GND
```

**สรุปขา I2C ของ OLED:**
| OLED | ESP32-C3 |
|------|---------|
| VCC | 3V3 |
| GND | GND |
| SCL | GPIO 1 ← ⚠️ เปลี่ยนจาก GPIO 6 (อยู่ใกล้ SPI Flash) |
| SDA | GPIO 0 ← ⚠️ เปลี่ยนจาก GPIO 7 (อยู่ใกล้ SPI Flash) |

> 💡 **เคล็ดลับ:** I2C สามารถต่ออุปกรณ์หลายตัวที่ใช้ I2C พร้อมกันได้ เพราะแต่ละตัวมี Address ต่างกัน

#### ขั้นตอนที่ 3: เขียนโค้ด

```cpp
// ============================================
// บทที่ 9: เครื่องวัดระยะแสดงผลบน OLED
// ============================================
// วัดระยะด้วย HC-SR04 และแสดงผลบนจอ OLED 0.96"

#include <Arduino.h>
#include <Wire.h>
#include <Adafruit_GFX.h>
#include <Adafruit_SSD1306.h>

// ==== กำหนดขา ====
#define TRIG_PIN   3   // HC-SR04 Trig → GPIO 3
#define ECHO_PIN   10  // HC-SR04 Echo → GPIO 10 (ผ่าน Voltage Divider)

// ==== กำหนดขนาด OLED ====
#define SCREEN_WIDTH  128  // จำนวนพิกเซลแนวกว้าง
#define SCREEN_HEIGHT 64  // จำนวนพิกเซลแนวสูง

// ==== สร้างอ็อบเจ็กต์ OLED ====
// Address I2C ของ SSD1306 คือ 0x3C
Adafruit_SSD1306 display(SCREEN_WIDTH, SCREEN_HEIGHT, &Wire, -1);

// ==== ตัวแปร ====
float duration_us = 0;   // เวลาที่เสียงใช้ (μs)
float distance_cm = 0;   // ระยะทาง (cm)

void setup() {
  Serial.begin(115200);
  Serial.println("=== HC-SR04 + OLED Distance Meter ===");

  // ตั้งค่า HC-SR04
  pinMode(TRIG_PIN, OUTPUT);  // Trig = Output
  pinMode(ECHO_PIN, INPUT);    // Echo = Input

  // เริ่ม OLED
  Wire.begin(0, 1);  // SDA=GPIO0, SCL=GPIO1 ← ⚠️ เปลี่ยนจาก GPIO 6-7 (อยู่ใกล้ SPI Flash)

  if (!display.begin(SSD1306_SWITCHCAPVCC, 0x3C)) {
    Serial.println("OLED ไม่พบ!");
    while (1);  // หยุดทำงานถ้าจอไม่เจอ
  }

  display.clearDisplay();     // ล้างจอ
  display.setTextColor(SSD1306_WHITE);  // ตั้งสีข้อความเป็นขาว
  display.setTextSize(2);      // ตั้งขนาดตัวอักษร
  display.display();

  Serial.println("พร้อมวัดระยะ!");
}

void loop() {
  // ===== ขั้นตอนที่ 1: ส่ง Trigger Signal =====
  digitalWrite(TRIG_PIN, LOW);   // เริ่มจาก LOW
  delayMicroseconds(2);           // รอ 2μs

  digitalWrite(TRIG_PIN, HIGH);  // ส่ง HIGH
  delayMicroseconds(10);          // ส่ง 10μs (ตาม spec)
  digitalWrite(TRIG_PIN, LOW);   // กลับเป็น LOW

  // ===== ขั้นตอนที่ 2: วัดเวลา Echo =====
  // pulseIn จะรอจนกว่า Echo ขึ้น HIGH แล้ววัดเวลาจนกลับ LOW
  duration_us = pulseIn(ECHO_PIN, HIGH);

  // ===== ขั้นตอนที่ 3: คำนวณระยะทาง =====
  // ระยะ (cm) = เวลา (μs) / 58
  // (ใช้ 58 เพราะ ความเร็วเสียง 343m/s = 0.0343cm/μs, หาร 2 = 58.1)
  distance_cm = duration_us / 58.0;

  // ตรวจสอบว่าระยะอยู่ในช่วงที่วัดได้ (2cm - 400cm)
  if (distance_cm < 2.0) {
    distance_cm = 2.0;
    Serial.println("⚠️  ใกล้เกินไป! (< 2cm)");
  } else if (distance_cm > 400.0) {
    distance_cm = 400.0;
    Serial.println("⚠️  ไกลเกินไป! (> 400cm)");
  }

  // ===== แสดงผลบน Serial Monitor =====
  Serial.print("ระยะ: ");
  Serial.print(distance_cm);
  Serial.println(" cm");

  // ===== แสดงผลบน OLED =====
  display.clearDisplay();           // ล้างจอ
  display.setTextSize(2);           // ตั้งขนาดใหญ่
  display.setCursor(10, 10);        // ตำแหน่งข้อความ
  display.print("Distance");

  display.setTextSize(3);           // ตั้งขนาดใหญ่มาก
  display.setCursor(20, 35);        // ตำแหน่งตัวเลข
  display.print(distance_cm, 1);   // แสดงทศนิยม 1 ตำแหน่ง

  display.setTextSize(2);            // ขนาดเล็กลง
  display.setCursor(95, 42);         // ตำแหน่งหน่วย
  display.print("cm");

  display.display();                 // แสดงผลจริง

  delay(200);  // วัดทุก 200ms (5 ครั้ง/วินาที)
}
```

#### ขั้นตอนที่ 4: ทดสอบ

1. **อัปโหลดโค้ด** ไปที่ ESP32-C3
2. **เปิด Serial Monitor** ที่ 115200 baud
3. **ยก HC-SR04 ขึ้น** แล้วใช้มือเข้า-ออกหน้าเซ็นเซอร์
4. **ดูระยะทางเปลี่ยนไป** บนจอ OLED และ Serial Monitor

**ผลลัพธ์ที่คาดหวัง:** 🎉
- จอ OLED แสดง "Distance: XX.X cm"
- ยิ่งมือใกล้เซ็นเซอร์ = ค่ายิ่งน้อย
- ยิ่งมือไกล = ค่ายิ่งมาก

> 💡 **เคล็ดลับ:** ถ้าค่ากระโดดไม่หยุด ให้ตรวจสอบว่า Voltage Divider ต่อถูกต้อง หรือลองเปลี่ยนสายเพราะอาจมีสัญญาณรบกวน

### 🛠️ ปฏิบัติที่ 2: ที่จอดรถอัตโนมัติ (ถ้าใกล้ → Buzzer เตือน) 🚗🔊

โปรเจกต์นี้เราจะทำระบบ **เตือนระยะจอดรถ** ที่จะ:
- เมื่อระยะ > 100cm → ไฟเขียว (ปลอดภัย) 🔋 ปกติ
- เมื่อระยะ 50-100cm → ไฟเหลือง + Buzzer สั่นช้า 🔶 เตือน
- เมื่อระยะ 20-50cm → ไฟส้ม + Buzzer สั่นเร็วขึ้น 🟠 ใกล้มาก
- เมื่อระยะ < 20cm → ไฟแดง + Buzzer สั่นเร็วมาก 🔴 หยุด!

#### ขั้นตอนที่ 1: ต่อวงจร

```
    ESP32-C3                    HC-SR04
    ┌─────────┐              ┌──────────┐
    │      GND├──────────────│○ GND     │
    │  5V    ├──────────────│○ VCC     │
    │  GPIO3 ├──────────────│○ Trig    │
    │  GPIO10◄─┤◄── R1+R2 ───│○ Echo    │
    └─────────┘              └──────────┘

    LED (ผ่าน R 220Ω แต่ละตัว):
    GPIO 18 ──── R220Ω ──── LED เขียว ──── GND
    GPIO 19 ──── R220Ω ──── LED เหลือง ──── GND
    GPIO 10 ──── R220Ω ──── LED ส้ม ──── GND
    GPIO 20 ──── R220Ω ──── LED แดง ──── GND

    Buzzer (Active):
    GPIO 21 ──────────────► (+) Buzzer (-) ──── GND
```

**สรุปการต่อ:**

| อุปกรณ์ | GPIO |
|--------|------|
| HC-SR04 Trig | 3 |
| HC-SR04 Echo (ผ่าน Voltage Divider) | 10 |
| LED เขียว | 18 |
| LED เหลือง | 19 |
| LED ส้ม | 10 |
| LED แดง | 20 |
| Buzzer | 21 |

#### ขั้นตอนที่ 2: เขียนโค้ด

```cpp
// ============================================
// บทที่ 9: ที่จอดรถอัตโนมัติ
// ============================================
// วัดระยะด้วย HC-SR04
// ถ้าใกล้ → Buzzer เตือน + LED เปลี่ยนสี

#include <Arduino.h>

// ==== กำหนดขา ====
#define TRIG_PIN   3   // HC-SR04 Trig
#define ECHO_PIN   10  // HC-SR04 Echo

// ⚠️ หมายเหตุ: GPIO 6-11 บน ESP32-C3 อยู่ใกล้ SPI Flash ภายใน
// ควรเลี่ยงใช้ GPIO เหล่านั้น ใช้ GPIO ที่ปลอดภัยกว่า เช่น GPIO 0, 1, 3, 10, 18, 19, 20, 21
#define LED_GREEN  18   // LED เขียว (ปลอดภัย)
#define LED_YELLOW 19   // LED เหลือง (เตือน)
#define LED_ORANGE 10   // LED ส้ม (ใกล้)
#define LED_RED    20   // LED แดง (อันตราย)
#define BUZZER_PIN 21  // Buzzer ← ✅ ปลอดภัย

// ==== ค่าตั้งต้น ====
#define SAFE_DISTANCE   100.0  // ระยะปลอดภัย (cm)
#define WARN_DISTANCE  50.0   // ระยะเตือน (cm)
#define CLOSE_DISTANCE 20.0   // ระยะใกล้มาก (cm)

void setup() {
  Serial.begin(115200);
  Serial.println("=== ที่จอดรถอัตโนมัติ ===");

  // ตั้งค่า HC-SR04
  pinMode(TRIG_PIN, OUTPUT);
  pinMode(ECHO_PIN, INPUT);

  // ตั้งค่า LED
  pinMode(LED_GREEN, OUTPUT);
  pinMode(LED_YELLOW, OUTPUT);
  pinMode(LED_ORANGE, OUTPUT);
  pinMode(LED_RED, OUTPUT);

  // ตั้งค่า Buzzer
  pinMode(BUZZER_PIN, OUTPUT);

  // เริ่มต้น: ไฟเขียวติด (ปลอดภัย)
  allLEDsOff();
  digitalWrite(LED_GREEN, HIGH);

  Serial.println("พร้อม! วัดระยะจอดรถ:");
  Serial.println(" >100cm = เขียว (ปลอดภัย)");
  Serial.println(" 50-100cm = เหลือง (เตือน)");
  Serial.println(" 20-50cm = ส้ม (ใกล้มาก)");
  Serial.println(" <20cm = แดง (หยุด!)");
}

void loop() {
  // ===== วัดระยะ =====
  float distance = measureDistance();

  // ===== ตรวจสอบสถานะ =====
  allLEDsOff();  // ปิดไฟทุกดวงก่อน

  if (distance > SAFE_DISTANCE) {
    // สีเขียว: ปลอดภัย
    digitalWrite(LED_GREEN, HIGH);
    noTone(BUZZER_PIN);  // ไม่มีเสียง
    Serial.print("🟢 ปลอดภัย: ");
    Serial.print(distance);
    Serial.println(" cm");

  } else if (distance > WARN_DISTANCE) {
    // สีเหลือง: เตือน
    digitalWrite(LED_YELLOW, HIGH);
    tone(BUZZER_PIN, 1000, 200);  // เสียง 1kHz, 200ms
    delay(300);
    Serial.print("🟡 เตือน: ");
    Serial.print(distance);
    Serial.println(" cm");

  } else if (distance > CLOSE_DISTANCE) {
    // สีส้ม: ใกล้มาก
    digitalWrite(LED_ORANGE, HIGH);
    tone(BUZZER_PIN, 1500, 150);  // เสียง 1.5kHz, 150ms
    delay(150);
    Serial.print("🟠 ใกล้มาก: ");
    Serial.print(distance);
    Serial.println(" cm");

  } else {
    // สีแดง: อันตราย! หยุด!
    digitalWrite(LED_RED, HIGH);
    tone(BUZZER_PIN, 2000);  // เสียงต่อเนื่อง 2kHz
    Serial.print("🔴 หยุด!: ");
    Serial.print(distance);
    Serial.println(" cm");
  }

  delay(100);  // วัดทุก 100ms
}

// ===== ฟังก์ชันวัดระยะ =====
float measureDistance() {
  // ส่ง Trigger
  digitalWrite(TRIG_PIN, LOW);
  delayMicroseconds(2);
  digitalWrite(TRIG_PIN, HIGH);
  delayMicroseconds(10);
  digitalWrite(TRIG_PIN, LOW);

  // วัดเวลา Echo
  long duration = pulseIn(ECHO_PIN, HIGH);

  // คำนวณระยะ (cm)
  float distance = duration / 58.0;

  // จำกัดค่า
  if (distance < 2.0) return 2.0;
  if (distance > 400.0) return 400.0;

  return distance;
}

// ===== ฟังก์ชันปิดไฟทุกดวง =====
void allLEDsOff() {
  digitalWrite(LED_GREEN, LOW);
  digitalWrite(LED_YELLOW, LOW);
  digitalWrite(LED_ORANGE, LOW);
  digitalWrite(LED_RED, LOW);
}
```

#### ขั้นตอนที่ 3: ทดสอบ

1. **อัปโหลดโค้ด** ไปที่ ESP32-C3
2. **เปิด Serial Monitor** ที่ 115200 baud
3. **ยกมือเข้า-ออก** หน้า HC-SR04 แล้วสังเกต:
   - มือไกล (>100cm) → LED เขียวติด
   - มืออยู่กลาง (50-100cm) → LED เหลืองติด + เสียงเตือนช้า
   - มือใกล้ (20-50cm) → LED ส้มติด + เสียงเตือนเร็ว
   - มือใกล้มาก (<20cm) → LED แดงติด + เสียงดังต่อเนื่อง!

**ผลลัพธ์ที่คาดหวัง:** 🎉
- ยิ่งเข้าใกล้ → ไฟยิ่งเปลี่ยนเป็นสีร้อน (เขียว → เหลือง → ส้ม → แดง)
- เสียง Buzzer ยิ่งถี่ขึ้นเมื่อใกล้ขึ้น
- ถ้าเป็นรถจอดจริง คนขับจะรู้ได้ทันทีว่าต้องหยุดตรงไหน!

> 💡 **เคล็ดลับ:** ถ้าต้องการติดตั้งจริงที่รถ ให้ติด HC-SR04 ไว้ที่กันชนหลัง แล้วต่อสายยาวเข้าห้องคนขับ อย่าลืมใช้สายที่มีชีลด์ (shielded cable) เพื่อป้องกันสัญญาณรบกวนจากเครื่องยนต์!

## 🔬 เรื่องพิเศษ: HC-SR04 vs อื่นๆ — เลือกใช้อะไรดี? 📊

HC-SR04 เป็นเซ็นเซอร์ยอดนิยม แต่มีทางเลือกอื่นๆ ด้วย:

| เซ็นเซอร์ | ระยะ | ความแม่นยำ | ราคา | ข้อดี |
|-----------|------|-----------|------|------|
| **HC-SR04** | 2-400cm | ±3mm | ถูก | ใช้ง่าย แม่นพอ |
| **US-100** | 2-450cm | ±2mm | ปานกลาง | มี Temperature Compensation |
| **VL53L0X** (Laser) | 0-200cm | ±1mm | แพง | ใช้ Laser แม่นมาก, วัดเร็ว |
| **TF-Luna** (LiDAR) | 0-400cm | ±3cm | แพงมาก | ใช้ LiDAR วัดได้ไกลและเร็ว |

```
📌 ข้อจำกัดของ HC-SR04:
1. วัตถุที่มีพื้นผิวเรียบและแข็ง (กระจก, กระเบื้อง) → สะท้อนดี
2. วัตถุที่ดูดซับเสียง (ฟองน้ำ, ผ้า, พรม) → สะท้อนไม่ดี
3. วัตถุที่เป็นมุม → เสียงอาจกระเจิงไม่สะท้อนกลับตรง
4. วัดผ่านน้ำไม่ได้ (เสียงดูดซับในน้ำมาก)
5. วัดต่อเนื่องเร็วเกินไป → เสียงก่อนอาจรบกวนเสียงหลัง
```

### แบบฝึกที่ 2: วัดระยะ 2 จุด
ต่อ HC-SR04 2 ตัว วัดระยะด้านหน้าและด้านหลังรถพร้อมกัน แล้วแสดงผลบน OLED

### แบบฝึกที่ 3: บันทึกค่าวัดระยะลง EEPROM
ให้ระบบบันทึกค่าระยะที่วัดได้ลง EEPROM ทุก 1 วินาที แล้วแสดงค่าเฉลี่ยเมื่อกดปุ่ม

### แบบฝึกที่ 4: หุ่นยนต์หลบหลีกสิ่งกีดขวาง (Line Follower รุ่นพิเศษ)
ใช้ HC-SR04 ติดหน้าหุ่นยนต์ 2 ตัว (ซ้าย-ขวา) เมื่อเจอสิ่งกีดขวางด้านหน้าให้หุ่นยนต์หลบไปทางที่ว่าง

### แบบฝึกที่ 5: วัดระยะ + เก็บข้อมูล
ให้ระบบวัดระยะทุก 1 วินาที แล้วแสดงกราฟบน OLED (ใช้ `display.drawLine()`)

## 📚 สรุป

ในบทนี้เราได้เรียนรู้ว่า:

✅ **HC-SR04 วัดระยะด้วย Ultrasound** โดยส่งเสียง 40kHz ออกไปแล้วรอฟังเสียงสะท้อน (Sonar)  
✅ **สูตรคำนวณระยะ = เวลา(μs) / 58** เพราะความเร็วเสียงในอากาศประมาณ 343m/s  
✅ **Echo Pin ส่งสัญญาณ 5V ออกมา** ต้องใช้ Voltage Divider ลดเป็น 3.3V ก่อนต่อเข้า ESP32-C3  
✅ **สามารถทำเครื่องวัดระยะ** แสดงผลบน OLED ได้โดยใช้ `pulseIn()` วัดเวลา  
✅ **ทำระบบเตือนจอดรถอัตโนมัติ** ได้โดยใช้ HC-SR04 + LED + Buzzer แสดงสถานะตามระยะที่วัดได้  

> 🔮 **บทต่อไป:** ในบทถัดไปเราจะมาเรียนรู้เรื่อง **Data Logging** กัน! เราจะได้รู้ว่าจะเก็บข้อมูลจากเซ็นเซอร์หลายตัวลง SD Card แล้วนำมาวิเคราะห์ด้วย AI ได้ยังไง! 📊🤖
