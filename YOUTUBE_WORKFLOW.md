# Workflow: สร้างคอร์ส YouTube ESP32-C3

## สรุป Workflow ฉบับเต็ม

### ขั้นตอนที่ 1: หนูเขียน Script
```
├── เขียน Script ทุกบท
├── กำหนดว่าหน้าไหน = Slide / AI Video / INSERT
├── หน้า Slide: มีบทพูด
├── หน้า AI Video: มี Prompt สำหรับ Kling/Veo
└── หน้า INSERT: มี TODO + บทพูดที่จะพูด
```

### ขั้นตอนที่ 2: หนูสร้าง
```
PowerPoint:
├── ทุกหน้ามีเสียงบรรยายในตัว
├── หน้า Slide: ภาพ + เสียง
├── หน้า AI Video: Prompt + placeholder
└── หน้า INSERT:
    ├── 📹 INSERT FOOTAGE
    ├── ชื่อหัวข้อ
    ├── ความยาว: ~30 วินาที
    ├── TODO: 1, 2, 3...
    ├── บทพูด (พี่พูดตามนี้)
    ├── ไฟล์เสียง: ch01_insert01.mp3
    └── ไฟล์ video: ch01_insert01.mp4 (placeholder)
```

### ขั้นตอนที่ 3: หนูสร้าง Video (Pending)
```
Video ที่ได้:
├── ความยาว: ~10-15 นาที/บท
├── มีเสียงบรรยายตลอดทั้งคลิป
├── หน้า Slide: เสียง + ภาพ slide
├── หน้า AI Video: เสียง + placeholder (รอ Kling/Veo)
└── หน้า INSERT: เสียง + คลิป placeholder เงียบ
```

### ขั้นตอนที่ 4: พี่ทำ
```
พี่ได้รับ:
├── PowerPoint (มีเสียงในตัว)
├── Video Pending (มีทุกอย่าง ยกเว้น INSERT จริง)
└── AI Prompts (สำหรับสร้าง AI Video)

พี่ต้องทำ:
├── ดู PowerPoint → รู้ว่าแต่ละหน้าต้องทำอะไร
├── ถ่าย Real Footage → ตาม TODO ในแต่ละ INSERT
│   • พูดตามบทพูดที่เขียนไว้
│   • ความยาวตามที่กำหนด
├── ตัดต่อใน Premiere/DaVinci
│   • แทนที่ INSERT Placeholder ด้วย Footage จริง
│   • เสียงบรรยายมีอยู่แล้ว → ไม่ต้องอัดใหม่
└── ส่งออก → Video สำเร็จ!
```

## Template หน้า Slide แบบ Simple

### หน้าอุปกรณ์:
```
┌──────────────────────────────────────────────────┐
│           📷 อุปกรณ์ที่ใช้                      │
│                                                  │
│  ┌─────────────┬────────────┬─────────────┐   │
│  │ อุปกรณ์    │ จำนวน     │ หมายเหตุ   │   │
│  ├─────────────┼────────────┼─────────────┤   │
│  │ ESP32-C3    │ 1 บอร์ด   │             │   │
│  │ LED         │ 1 ดวง     │ สีแดง       │   │
│  │ Resistor    │ 1 ตัว     │ 220Ω        │   │
│  │ Breadboard  │ 1 อัน     │             │   │
│  │ Jumper Wire │ 3 เส้น    │             │   │
│  └─────────────┴────────────┴─────────────┘   │
└──────────────────────────────────────────────────┘
```

### หน้าวงจร:
```
┌──────────────────────────────────────────────────┐
│           📷 การต่อสาย                       │
│                                                  │
│  ESP32-C3        →        LED Circuit           │
│  ─────────────────────────────────────          │
│  GPIO 8          →        Resistor → LED (+)    │
│  GND              →        LED (-)               │
│                                                  │
│  สรุป: GPIO 8 → R220Ω -> LED -> GND             │
└──────────────────────────────────────────────────┘
```

### หน้าโค้ดตัวอย่าง:
```
┌──────────────────────────────────────────────────┐
│           💻 โค้ดตัวอย่าง                      │
│                                                  │
│  #define LED_PIN 8    // LED ของ ESP32 C3      │
│                                                  │
│  void setup() {                                │
│    pinMode(LED_PIN, OUTPUT);                   │
│  }                                             │
│                                                  │
│  void loop() {                                 │
│    digitalWrite(LED_PIN, LOW);  // LED ติด     │
│    delay(1000);                                │
│    digitalWrite(LED_PIN, HIGH); // LED ดับ     │
│    delay(1000);                                │
│  }                                             │
│                                                  │
│  📁 โค้ดเต็ม: chapter03/blink_led.ino         │
└──────────────────────────────────────────────────┘
```

## หน้าที่หนูสร้างเอง vs ต้องรอ INSERT

### ✅ หนูสร้างเอง:
- 📷 หน้าอุปกรณ์ (ตารางรายชื่อ)
- 📷 หน้าวงจร (บอกว่าขาไหนต่อขาไหน)
- 📷 หน้าโค้ดตัวอย่าง (code บางส่วน + ลิงค์)

### ⏳ ต้องรอ INSERT (พี่ถ่ายจริง):
- 📹 หน้าต่อวงจรจริง
- 📹 หน้าผลลัพธ์จริง
