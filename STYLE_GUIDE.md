## 🎯 สไตล์การเขียน

### โทนเสียง: "เพื่อนคุยกันแบบสุภาพ"
- ใช้คำว่า "เรา" แทน "เธอ" หรือ "มึง"
- สนุก เป็นกันเอง แต่ไม่ล่วงเกิน
- อธิบายเหมือนเพื่อนที่รู้มากกว่าช่วยเพื่อนที่รู้น้อยกว่า
- ไม่ใช้คำหยาบ ไม่ดูถูก เคารพผู้อ่าน

### ตัวอย่างโทน:
✅ "เรามาดูกันว่า LED ทำงานยังไงนะ"
✅ "เวลาที่เรากดปุ่ม ไฟจะติด ง่ายๆ เลย"
❌ "มึงจะใช้ GPIO ได้ยังไง นี่มึงไม่รู้เลยเหรอ?"

- คำศัพท์เทคนิคภาษาอังกฤษ: อธิบายความหมายทุกครั้ง
  - เช่น "GPIO (General Purpose Input/Output) - ขาที่ใช้รับ-ส่งสัญญาณ"

### Emoji
- ใช้พอเหมาะ (บทนำ, หัวข้อ, เนื้อหาสำคัญ)
- ไม่ใช้เกิน 2-3 ตัวต่อย่อหน้า

## 📖 โครงสร้างแต่ละบท

### 1. บทนำ (Introduction)
- บอกว่าบทนี้จะเรียนอะไร
- ทำไมต้องรู้ (ให้เหตุผล)
- มีตัวอย่างในชีวิตจริง

### 2. อุปกรณ์ที่ใช้ (Equipment)
- รายการอุปกรณ์พร้อมรูปภาพ
- ราคาโดยประมาณ

### 3. เนื้อหา (Content)
- อธิบายทฤษฎีแบบเข้าใจง่าย
- มีภาพประกอบ (ถ้าทำได้)
- ยกตัวอย่างในชีวิตจริง

### 4. ปฏิบัติ (Practice)
- ขั้นตอนทำทีละขั้น
- โค้ดพร้อม comment อธิบาย
- วิธีตรวจสอบว่าสำเร็จ

### 5. แบบฝึก (Exercise)
- คำถามทบทวน (3-5 ข้อ)
- โจทย์ปัญหา
- กิจกรรมเพิ่มเติม

### 6. สรุป (Summary)
- สรุปสิ่งที่เรียนรู้
- เชื่อมต่อกับบทถัดไป

## 📝 รูปแบบโค้ด

```cpp
// ตัวอย่างโค้ดพร้อม comment ภาษาไทย
#include <Arduino.h>

void setup() {
    // ตั้งค่า LED ที่ขา 2 เป็น output
    pinMode(2, OUTPUT);
}

void loop() {
    // สั่งให้ LED ติด
    digitalWrite(2, HIGH);
    delay(1000); // รอ 1 วินาที
    
    // สั่งให้ LED ดับ
    digitalWrite(2, LOW);
    delay(1000); // รอ 1 วินาที
}
```

3. ตรวจสอบขั้วบวก-ลบให้ถูกต้อง
